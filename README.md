# 🧪 CraftersLab QA: Web Application & Browser Extension Testing

This repository documents a complete Software Quality Assurance (SQA) cycle for the **CraftersLab web platform and browser extension**, including requirement analysis, test planning, execution, and defect reporting.

---

## 🔗 Application Under Test

- 🌐 Web Application: http://app.crafterslabsoft.com/dashboard  
- 🧩 Browser Extension: Internal Testing Build (Not Public)

---

## 🎯 Objective

To ensure the reliability, functionality, and usability of the CraftersLab platform by performing structured manual testing based on requirements and real-world user behavior.

---

## 📌 Scope of Testing

- Functional Testing  
- UI/UX Validation  
- Cross-Browser Compatibility Testing  
- Regression Testing  
- Exploratory Testing  
- Negative Testing  

---

## 📄 QA Deliverables

This repository includes the full QA lifecycle documentation:

- 📘 SRS Analysis (Requirement Breakdown & Test Mapping)  
- 🧪 Test Plan (Strategy, Scope, Approach, Environment)  
- ✅ Test Cases (Positive & Negative Scenarios)  
- 🐞 Bug Reports (Repro Steps, Severity, Priority, Evidence)  
- 📸 Screenshots (Issue Proofs where applicable)  

---

## 🛠 Tools Used

- Manual Testing Approach  
- Google Chrome DevTools  
- Jira / Issue Tracking System  
- Microsoft Excel / Google Sheets (Test Case Management)  
- Microsoft Word (Bug Documentation)  

---

## 📊 Key Highlights

- Built structured test cases from raw SRS requirements  
- Identified functional and UI inconsistencies across web & extension  
- Documented reproducible bug reports with clear steps  
- Validated system behavior across multiple test scenarios  
- Improved product stability through systematic QA execution
